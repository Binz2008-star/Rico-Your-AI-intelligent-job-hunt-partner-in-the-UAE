Two references, one honest idea:
reveal the clarity that's already there.

The Lithos spotlight and the Mainframe pointer-scrub are motion techniques, not templates. Extracted and rebuilt in Rico's Atelier identity — warm paper, ink, clay — the spotlight reveals an organised career state beneath a scattered one, and the scrub walks that same transformation. No neon, no glassmorphism, no geology, no fake data.

What we took
Lithos → spotlight reveal. Cursor-following soft radial mask over two stacked layers; smoothed pointer via RAF + lerp; slow Ken-Burns zoom; staggered blur-rise type.
Mainframe → pointer scrub. Horizontal pointer scrubs a short transformation sequence; one active seek + queued latest target; typewriter intro; staggered action-pill entrance; minimal nav.
What we changed for Rico
· The reveal shows clarity (active CV, verified matches, a structured board) — never a decorative second image.
· CSS radial-gradient mask via custom properties — not canvas.toDataURL() per frame.
· Pointer events (not mouse-only), touch fallback, keyboard-safe, reduced-motion static fallback.
· The scrub is a truthful SVG keyframe sequence, not fake dashboard footage.
· Atelier palette + editorial type; every sample carries the “not live account data” label.

// useSmoothPointer (abridged)
const tick = () => {
  sm.x += (raw.x - sm.x) * 0.12;
  sm.y += (raw.y - sm.y) * 0.12;
  el.style.setProperty('--spotlight-x', sm.x + 'px');
  el.style.setProperty('--spotlight-y', sm.y + 'px');
  raf = active ? requestAnimationFrame(tick) : 0;
};

09 · Non-negotiable

Performance & accessibility

Motion is a finish, not a gate. Everything degrades cleanly and nothing essential is locked behind a pointer.

GPU-only animation. Transform, opacity, and mask position — no layout thrash, will-change scoped.
No per-frame toDataURL(). The mask is a CSS gradient moved by custom properties.
Pointer events, not mouse-only. pointermove/leave/cancel; touch gets the static clear state.
Reduced motion honoured. Media query + a manual switch; RAF loops don't start.
Keyboard-safe. Hero/scrub are focusable with descriptive labels; arrows drive the seam; spotlight never traps focus.
Text stays selectable. Overlays are pointer-events:none; the Command spotlight disables on first interaction.
RAF self-parks. The loop cancels when idle and on unmount; no background CPU burn.
Mobile-safe height. 100dvh so browser chrome never clips the hero.
SSR-safe. All window/matchMedia reads are guarded for Next.js.
Truthful throughout. No fake counts, employers, statuses, verification, or progress; samples labelled.
